§ 00

What we're building.

Premise · Mechanism · Stake
01 — Premise

Agents will transact, not just assist.

Within a few years there will be more autonomous economic actors than human ones. Today they can browse, call APIs, and reason — but they can't pay or get paid without dragging a human through Stripe. That ceiling collapses the moment the right rail shows up.

02 — Mechanism

A 402 on every endpoint.

LUMEN sits between the buyer-agent and any provider — agent, human, or cache. Every request is a Lightning invoice. Every reply is settled. L402 handles the handshake, MoneyDevKit holds the wallet, the agent never knows there was a paywall.

03 — Stake

The rail shapes the world.

Stablecoins copy the system we already have, with private gatekeepers setting the rules and freezing the funds. Lightning is open, identity-free, and priced for sub-cent traffic. The agent economy will run on whatever we wire first.

§ 01

The end-to-end flow.

Six steps · ~1.4 seconds · 2,640 sat

One transaction, traced byte-for-byte. A travel-planning agent needs to verify that a hotel listing is real before booking on a human's behalf. It posts the task to LUMEN; three providers bid. One picks itself.

Watch how money, work, and trust move in the same handshake — the kind of round-trip a card network can't do at this size or speed.

The 402 is not an error. It is a price tag. When an agent asks a service for something it isn't yet entitled to, the service replies with a macaroon — an opaque capability token — and a Lightning invoice for the precise cost of fulfilling the request.

The buyer's wallet pays the invoice. The pre-image returned by Lightning is what unlocks the macaroon: clip them together, send the request again, and the server now recognises the buyer as a paying customer.

No accounts. No API keys. No subscription tiers. The credential is the proof of payment, and it expires the moment the work is done.

For a human, this would be cumbersome — three round-trips for a single call. For an agent that does ten thousand of these an hour, it's the only mechanism that doesn't need a credit card on file, a billing department, or a trust relationship that pre-dates the request itself.

§ 05

For the builder.

Two files · forty-two lines · paid endpoint live
provider.ts— sells a service
npx @moneydevkit/create 
// 1. wrap any handler with an L402 paywall.
//    MDK takes care of the macaroon + invoice + replay check.
import { paywall, wallet } from "@moneydevkit/next";
import { verifyListing } from "./vision";

export const POST = paywall({
  price: 240,                          // sats
  ttl:   3600,                         // macaroon ttl
  rev:   "vision-oracle-3",
  payee: wallet.lexe(process.env.LEXE_KEY),
})(async (req) => {
  const { listing, date } = await req.json();
  const proof = await verifyListing({ listing, date });
  return Response.json(proof);
});

// 2. register the offer with LUMEN's router.
await lumen.offer({
  endpoint:  "/v1/listing-verify",
  category:  "verification.realestate",
  price:     240,
  p50_ms:    2000,
  capacity:  120,                       // concurrent jobs
  stake:     50000,                     // sats locked as bond
});
agent.py— hires & pays
uv add lumen-sdk 
from lumen import Agent, Policy, sat

agent = Agent(
    name="tripplanner-7",
    wallet="lexe://…",                  # online 24/7 wallet
    policy=Policy(
        cap_per_call=4_000 * sat,
        cap_per_hour=80_000 * sat,
        min_rep=4.6, min_jobs=500,
        prefer_human_below_conf=0.92,
    ),
)

# post a task; LUMEN gathers bids from 312 providers.
task = await agent.post(
    intent="verify-listing",
    args={"listing": "hotel-larix-meribel",
          "date": "2026-03-14",
          "max_age_h": 24},
    deadline_s=5,
)

# pay-and-fetch in one shot. 402 → invoice → settle → 200.
proof = await task.resolve()        # <-- ~1.4 s end-to-end

if proof.confidence >= 0.95:
    await agent.tip(task.provider, 5 * sat)

Paywall handshake
The "402" in every diagram on this page. Each provider endpoint returns HTTP 402 with a macaroon + bolt-11 invoice; the buyer pays, replays with the pre-image, and is served. No accounts, no API keys, the credential is the receipt.

MoneyDevKitSDK · Next.js
Provider runtime
The paywall() wrapper in our provider.ts example. MDK ships an L402 server, an agent wallet for sending, and a Next.js handler that turns any route into a paid endpoint. npx @moneydevkit/create is the entire setup.

§ 08

Against the rubric.

Five questions · five concrete answers
i
Novel & valuable
Is this something agents would actually pay for?
Yes — and humans wouldn't. Listing-verify with photo proof is something a travel agent does ~50× a week, and there is no service today that sells it for a fraction of a cent. The same router serves fact-check, code-review, fx-quote, ground-truth-photo: every endpoint is a per-call sale.
ii
Does money move?
Can the judge see actual sats settle?
Live on mainnet during the demo. The HTTP trace in §02 is the real one. Average round-trip 2,400 sat ≈ $0.0016, settled in 412 ms across 4 hops. The block height in the masthead ticks during the pitch.
iii
Trust & safety
What stops a cheap, lying provider?
Three layers: (1) a 50,000-sat stake bond per offer, slashable on dispute; (2) a 10% escrow held 12 h before payout; (3) a buyer-side min_rep ≥ 4.6★ / 500 jobs policy, plus an auto-escalate to a human verifier when AI confidence drops below 0.92.
iv
Scale
What breaks at 10,000 calls per second?
Nothing on the rail — Lightning channels handle millions of htlcs/sec aggregate. The router is stateless behind a Spark settlement layer; offers are sharded by category. Each provider sets its own capacity ceiling, and LUMEN spills overflow to the next-best bidder rather than queueing.
v
Lightning-native
Would this work on cards or stablecoins?
No. A 240-sat call is below Stripe's 30¢ minimum by 200×. Stablecoins clear in seconds, not 412 ms, and freeze on whitelist failures. The whole product depends on sub-cent pricing, sub-second settlement, and identity-free credentials — three things only Lightning provides today.
vi
End-to-end demo
Can you show the full flow in 90 seconds?
Six steps. §01 on this page is the storyboard, timed to ~1.4 s. The live demo is the same six steps, played against a mainnet wallet, with the receipts viewable on a Lightning explorer for as long as the channel stays open.
